    Preguntas

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    • Esto nos ayuda a comprender si se esta ejecutando todo de manera correcta en especial algunas operaciones. En otras palabras nos ayuda a verificar el flujo del codigo para evitar errores.

  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    • Una de las formas es desactivar javascript del navegador y con eso se puede bypass la seguridad de javascript. Tambien se puede inspeccionar la pagina y obtener la informacion requerida (si no esta protegida). Referencia: https://developer.mozilla.org/en-US/docs/Web/Security

  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    • Porque es una buena practica ya que ayuda a evitar errores en el flujo del programa y por ende no enviar datos que puedan dañar nuestro servidor (o volverlo mas lento). Tambien es para agregar mas seguridad ya que no queremos que nuestro servidor sea objeto de algun ataque y lo mas importante proteger los datos del usuario.
